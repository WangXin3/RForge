## 1. 项目背景与目标
构建一个集“知识查阅”与“知识考核”为一体的智能系统。用户既能通过私有文档获取答案，也能通过系统的主动提问来检验对知识点的掌握程度。

## 2. 用户角色
* 普通用户： 上传私有文档、进行 RAG 问答、参与知识考核。
* 管理员： 维护平台公共知识库，监控系统运行状态。

## 3. 核心功能需求

| 模块         | 功能点         | 描述                                                   |
| ------------ | -------------- | ------------------------------------------------------ |
| 知识库管理   | 文档上传与解析 | 支持 PDF/Word，自动进行切片处理并存入数据库。          |
|              | 库权限隔离     | 区分“个人私有库”与“平台公共库”，确保数据隐私。         |
| 智能问答模式 | 增强检索问答   | 用户提问，系统检索相关片段并生成带有引用的回答。       |
| 智能考核模式 | 自动命题       | 系统随机从选定知识库中抽取内容，生成简答题或思考题。   |
|              | 智能判分       | 用户回答后，LLM 对比原文进行准确度打分，并指出遗漏点。 |

## 4. 成功指标 (KPI)
* 检索准确率： 关键信息召回率 > 90%。
* 考核反馈时延： 系统判分反馈控制在 3 秒以内（使用流式传输）。
